Módulo: Git

O que é Git?

Git é um sistema de controle de versão distribuído, gratuito e de código aberto, projetado para gerenciar o histórico de código de projetos de software. Ele permite que múltiplos desenvolvedores trabalhem no mesmo projeto simultaneamente de forma organizada, registrando alterações, revertendo erros e mantendo o histórico de evolução seguro.

Aulas Básicas

  1. Configuração Inicial: Configurando nome e email global com git config --global user.name.
  2. Inicialização: Como criar um repositório local usando git init.
  3. Fluxo de Trabalho (Working Directory, Staging Area, Repository): Entender a diferença entre arquivos modificados, preparados e commitados.
  4. Preparação e Registro: Usar git add . para preparar os arquivos e git commit -m "mensagem" para salvar as alterações.
  5. Rastreamento: Verificar o status dos arquivos com git status e o histórico de commits com git log.
  6. Ramificação (Branching): Criar ramificações com git branch [nome-da-branch] e alternar entre elas para testar novos recursos com segurança.

Curiosidades

  • 📅 O Git foi criado por Linus Torvalds (o criador do Linux) em 2005. Ele escreveu a primeira versão em apenas duas semanas!
  • 🧠 Linus criou o Git porque a comunidade do Linux perdeu o direito de usar o software BitKeeper gratuitamente.
  • 💬 Na gíria britânica, a palavra "Git" significa uma "pessoa desagradável" ou "tola". Linus brincou dizendo que batizou o sistema em sua própria homenagem.
  • 🔒 O Git usa algoritmos criptográficos (SHA-1) para identificar e garantir a integridade dos commits, tornando quase impossível alterar o histórico sem ser detectado.
  • 🌍 Diferente de sistemas de controle de versão antigos, o Git é distribuído: cada desenvolvedor tem uma cópia local completa de todo o histórico do projeto no seu computador.
